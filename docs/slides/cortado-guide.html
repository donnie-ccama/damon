01 · the idea

What is Cortado?

Cortado lets you create named AI assistants ("agents") that live on your computer. Each one works in its own window, on its own copy of a project — and remembers what it learned the next time you talk to it.

◆

Named agents

Not one anonymous chatbot — a roster. "Scout" researches, "Fixer" fixes the website. You open them by name.

✎

Real memory

Each agent keeps its own notes about itself, about you, and about its work. Come back tomorrow — it still knows.

⌂

Own workspace

Every agent gets a private copy of its project, so it can work without disturbing yours.

01 · the idea

Think of it as hiring a small team

  • Agents belong to teams. A team is just a group — newsletter, web — and each agent has a full name like newsletter/scout.
  • Each agent has a role. A one-line job description you write when you create it, like "Researches topics for the newsletter."
  • Short names work. If only one agent is called Scout, cortado open scout is enough.
  • They keep working between visits. Closing an agent's window doesn't fire it — the session keeps running in the background until you end it.

02 · how it works

Deliberately boring under the hood

No cloud. No database. No hidden background service.
Cortado glues together three simple, trusted pieces:

🗂

Plain folders

Everything Cortado knows lives in ordinary files under ~/cortado. You can read them, edit them, and back them up like any documents.

⧉

tmux

A battle-tested tool that keeps terminal sessions alive in the background — it's why your agents survive closed windows.

▣

Ghostty

The terminal window where you meet your agent. Cortado opens one for you; inside, the AI (Claude Code) is already running.

Because nothing is cached or hidden, what Cortado shows you is always the truth of what's on disk.

02 · how it works

An agent is just a folder

~/cortado/teams/newsletter/agents/scout
scout/ ├── agent.toml who it is: name, role, which AI runs it ├── memory/ its notebook — survives everything │ ├── AGENT.md "who am I" │ ├── USER.md "what I know about you" │ └── MEMORY.md "what I've learned so far" ├── worktree/ its private copy of the project └── logs/ history of every session

Delete the folder → the agent is gone. Copy the folder to another machine → the agent moves, memory and all.

02 · how it works

Memory that actually lasts

  • The agent writes its own notes. At the end of a session it reflects on what it learned and updates its markdown notebook.
  • Memory lives outside the project files, so switching branches or resetting code never erases what the agent knows.
  • It's plain text you can read. Peek any time with cortado memory newsletter/scout, or edit it yourself with --edit.
  • First open only: Claude Code asks permission to load these external memory files — answer allow. That's by design.

03 · using it

One-time setup

Install once, then forget about it. (Full copy-paste commands are in the README.)

1 · install the basicsgit, tmux, Ghostty, and the Claude Code AI runtime
2 · cortado doctorchecks everything is in place — and tells you exactly what's missing and how to install it
3 · cortado initcreates the ~/cortado home folder and a default config

Works identically on macOS and Linux. If doctor is happy, you're ready.

03 · using it

Your first agent — three commands

 terminal
$ cortado team new Newsletter $ cortado agent new newsletter/Scout --repo-new --role "Researches topics" $ cortado open scout session cortado_newsletter_scout_1 ← a window opens; say hi to Scout
  • --repo-new gives Scout a brand-new empty project to work in.
  • Already have a project? Use --repo-worktree ~/Projects/my-site — the agent works on its own copy, on its own branch, and never touches your files.

03 · using it

The four commands you'll use daily

cortado open scoutopen (or return to) an agent — picks up exactly where you left off
cortado sessionssee which agents are alive right now
cortado kill scoutactually end an agent's sessions (closing the window does not)
cortado memory newsletter/scoutread the agent's notebook; add --edit to change it

Want two Scouts working in parallel? cortado open scout --new starts a second session of the same agent.

03 · using it

Close the window. Come back tomorrow.

These are safe ✓

Closing the Ghostty window · logging out · quitting the dashboard · walking away for a week. The session keeps running; cortado open reattaches exactly where it was.

These end a session ✗

Only cortado kill — or restarting your computer. Even then, the agent's memory is untouched: kill ends the conversation, never the knowledge.

Session vs. memory: a session is a running conversation. Memory is the notebook. Sessions come and go; the notebook stays.

03 · using it

Prefer a dashboard? cortado ui

One screen with your whole roster: teams and agents on the left (live agents get a ● badge), sessions and memory on the right — in this very Tokyo Night theme.

03 · using it

Choosing which AI powers an agent

  • Friendly names, not model IDs. A registry file (models.toml) maps names like claude, codex, or kimi to the right AI. Pick one with cortado open scout --model kimi.
  • Some models need an account key. Store it once, safely, in your computer's built-in keychain:
 terminal
$ cortado key set openrouter Enter key for 'openrouter': •••••••••••• ✓ saved to keychain

Keys are never written to a file — they live in the macOS Keychain (or the Linux equivalent) and are handed to the agent only for the life of its session.
